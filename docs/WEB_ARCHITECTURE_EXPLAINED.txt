========================================
WEB APPLICATION ARCHITECTURE EXPLAINED
========================================

HOW EVERYTHING WORKS TOGETHER:
===============================

┌──────────────────────────────────────────────────────────────┐
│  YOU RUN: python web\wsgi.py                                 │
│           ↓                                                  │
│  wsgi.py (Entry Point) - Starts the web server              │
│           ↓                                                  │
│  Imports app.py and runs it with Waitress server            │
└──────────────────────────────────────────────────────────────┘

DETAILED FLOW:
--------------

1. wsgi.py (16 lines) - THE STARTER
   ├─ Imports the Flask app from app.py
   ├─ Uses Waitress web server (production-grade)
   └─ Starts server on http://127.0.0.1:5000

2. app.py (860 lines) - THE BRAIN
   ├─ Creates Flask application
   ├─ Defines all routes (URLs):
   │   ├─ / → Dashboard
   │   ├─ /register → Student Registration
   │   ├─ /entry → Entry Gate Camera
   │   ├─ /exit → Exit Gate Camera
   │   ├─ /reports → Reports Page
   │   └─ /api/* → API endpoints for camera, data, etc.
   ├─ Handles business logic
   ├─ Connects to database
   ├─ Processes face recognition
   └─ Renders HTML templates

3. templates/*.html (7 files) - THE PAGES
   ├─ base.html - Master template (header, footer, navigation)
   ├─ dashboard.html - Home page with stats
   ├─ register.html - Student registration form
   ├─ entry.html - Entry gate camera page
   ├─ exit.html - Exit gate camera page
   ├─ reports.html - Attendance reports
   └─ student_attendance.html - Individual student records

4. static/css/style.css - THE STYLING
   └─ Makes pages look beautiful (colors, layout, animations)

5. static/js/*.js (6 files) - THE INTERACTIVITY
   ├─ main.js - Common functions, notifications
   ├─ entry.js - Entry gate camera logic
   ├─ exit.js - Exit gate camera logic
   ├─ register.js - Student registration logic
   ├─ reports.js - Report generation logic
   └─ student_attendance.js - Attendance viewing logic

6. static/images/ - THE GRAPHICS
   ├─ IGIT.png - College logo
   └─ MYCOMP.png - Society logo


HOW IT ALL CONNECTS:
=====================

USER VISITS: http://127.0.0.1:5000/entry
            ↓
wsgi.py → app.py finds route "/entry"
            ↓
app.py executes entry_page() function
            ↓
Renders templates/entry.html
            ↓
HTML loads static/css/style.css (styling)
            ↓
HTML loads static/js/entry.js (camera logic)
            ↓
User sees Entry Gate page in browser
            ↓
User clicks "Start Camera"
            ↓
entry.js sends camera frames to /api/recognize_entry
            ↓
app.py processes frame with face recognition
            ↓
Returns result to entry.js
            ↓
entry.js shows "Student Recognized" notification


SIMPLE ANALOGY:
===============

Think of it like a restaurant:

wsgi.py      = The front door (entry point)
app.py       = The kitchen (prepares everything)
templates/   = The menu (what customers see)
static/css/  = The decoration (makes it pretty)
static/js/   = The waiter (brings data back and forth)
src/         = The ingredients store (business logic)


FILE RELATIONSHIPS:
===================

wsgi.py
  └─ imports app.py
      ├─ imports src/* (database, face recognition, etc.)
      ├─ serves templates/*.html
      │    └─ which load static/css/*.css
      │    └─ which load static/js/*.js
      └─ provides API endpoints for JavaScript


WHAT YOU NEED TO KNOW:
======================

✓ You only run: python web\wsgi.py
✓ wsgi.py is just 16 lines - it's a launcher
✓ app.py has all the logic (860 lines)
✓ Templates are HTML pages with placeholders
✓ Static files (CSS/JS) make pages interactive
✓ JavaScript calls API endpoints in app.py


COMMON ROUTES IN app.py:
=========================

Page Routes (returns HTML):
  /                    → Dashboard
  /register            → Registration form
  /entry               → Entry camera
  /exit                → Exit camera
  /reports             → Reports page

API Routes (returns JSON data):
  /api/recognize_entry → Process entry camera frame
  /api/recognize_exit  → Process exit camera frame
  /api/register_faces  → Register new student
  /api/attendance      → Get attendance records
  /api/reports/daily   → Generate daily report


WHY THIS STRUCTURE?
===================

✓ Separation of concerns
  - wsgi.py: Server startup
  - app.py: Application logic
  - templates: User interface
  - static: Styling & interactivity

✓ Easy to maintain
  - Change UI: Edit templates & CSS
  - Change logic: Edit app.py
  - Change server: Edit wsgi.py

✓ Professional standard
  - This is how real-world Flask apps work


QUICK SUMMARY:
==============

wsgi.py     → Starts the server (RUN THIS)
app.py      → Handles all requests & logic
templates/  → HTML pages users see
static/     → CSS styling + JavaScript interactivity

That's it! Everything else is just details.
